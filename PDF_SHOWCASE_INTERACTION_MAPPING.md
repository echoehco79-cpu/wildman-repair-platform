# PDF 功能复现映射

PDF本身不能执行网页JavaScript、表单提交、自动播放和内存状态机，因此采用以下可验证的等效表达，不把静态页面伪装成仍可执行的网页。

| 网页功能 | PDF中的复现方式 |
| --- | --- |
| 01—12章节按钮 | 每页顶部章节号均为内部链接；PDF书签同步提供章节导航 |
| 章节总览 | 独立总览页，12个章节块可点击 |
| 上一页/下一页 | 使用PDF阅读器翻页；每页底部显示当前页码与章节总览链接 |
| 图片灯箱 | As-Is与To-Be各提供独立全幅放大页，仍可在阅读器内继续缩放 |
| 第06章流程节点点击 | 14个节点展开为“流程—产品承载索引”，每个节点可跳转对应产品模块 |
| 第08章机制切换 | 原始报修、AI、规则、系统、人工在一页并列展开 |
| 第09章模块切换 | 12个真实渲染状态各自独立成页，产品目录可点击切换 |
| 表单、按钮和下拉操作 | 展示固定输入、系统处理、规则结果、输出和下一责任人；不宣称PDF可提交表单 |
| 第09B数据表切换 | 8张表各自独立成页，顶部表名可点击切换 |
| 记录详情抽屉 | 每张表底部展开第一条记录的完整字段详情 |
| 第10章场景载入/上一步/下一步 | 每个场景展示最终状态、完整时间线、验证结果及后端数据变化 |
| 自动播放/暂停 | 转换为固定步骤链，不在PDF中伪造自动播放 |

## PDF内展开的产品模块

智能报修、AI分析、信息补充、责任路由、人工复核、统一工单、供应商协同、SLA异常管理、维修结果、门店验收、返修、设备履历。

## PDF内展开的数据表

Asset、FaultEvent、NotificationLog、ResponsibilityParty、RoutingRule、StateEvent、Store、WorkOrder。

## PDF内展开的验证场景

正常自动派单、信息缺失、低置信度、P1超时、验收返修。
